    Kimi Claw 与本地 OpenClaw 飞书接入教程¶

    标签：API配置：有 环境：本地 / 云均可 安全性：中 IM接入：飞书

    这篇文档整理自 tmp10。原稿其实包含两条路线：

    1. Kimi Claw 极简接入飞书，适合小白快速上手
    2. 本地 OpenClaw + Kimi API + 飞书，适合愿意自己部署的人

    我把它整理成一篇“先易后难”的教程，方便你按自己的情况选择。

    1. 先理解这套系统是怎么工作的¶

    原稿用一个很好的比喻解释了 OpenClaw 的结构：

    1. 大模型：负责思考和生成内容，比如 Kimi、Claude
    2. OpenClaw：像项目经理，负责调用工具和组织流程
    3. 飞书：你和机器人沟通的办公入口
    4. Gateway：负责把外部消息安全转发到内部 OpenClaw

    理解这层关系后，后面看到 ID、Secret、事件订阅、回调配置这些步骤时，就更容易知道它们各自是在给哪个环节授权。

    2. 为什么很多人会用 Mac mini 或云端部署¶

    原稿给了两个比较现实的理由：

    2.1 为什么有人买 Mac mini¶

    1. 新机器更干净，不怕把主力开发环境弄乱
    2. 可以长期开机，适合 24/7 运行
    3. 数据隔离比日常办公机更放心

    2.2 为什么有人推荐云上部署¶

    1. 可以 24/7 持续运行
    2. 和本地文件隔离，误删风险更低
    3. 不一定非要买额外硬件

    但云端的缺点也很明显：
    它天然离你的本地文件更远，所以不如“贴身助理”那样懂你的真实工作环境。

    3. 路线一：Kimi Claw 一步到位¶

    这是原稿最推荐的路线，适合：

    1. 零基础
    2. 想 10 分钟左右跑通
    3. 希望先看到效果，再决定是否本地部署

    3.1 打开 Kimi Claw¶

    Kimi 官网：

    https://www.kimi.com/

    原稿指出，Kimi 团队已经把一部分 OpenClaw 相关能力做成了更傻瓜化的配置方式。

    同时，原稿也提到 Minimax 也开始提供类似的极简路线。

    3.2 购买 Kimi 会员¶

    原稿使用的是 Allegretto 会员计划。

    3.3 先在 Kimi Claw 里试一轮¶

    先直接用它内置的 Kimi 模型做对话测试，确认服务可用。

    然后你也可以先设置一个简单的定时任务，感受它的工作方式。

    3.4 不会接飞书怎么办¶

    原稿这里的思路非常直接：

    1. 直接问 Kimi
    2. 在提示词里明确说“我是小白，请一步步教我”

    这条思路对初学者其实很实用，因为它能把配置步骤即时翻译成你能理解的话。

    4. 用 Kimi Claw 接入飞书¶

    4.1 打开飞书开发者后台¶

    飞书开放平台：

    https://open.feishu.cn/

    先创建企业自建应用。

    然后给应用起个名字。

    4.2 获取 App ID 和 Secret¶

    创建后，就可以在后台看到应用的 ID 和 Secret。

    把它们发给 Kimi Claw 进行绑定。

    如果它中途长时间没有反应，原稿建议可以尝试重启实例。

    4.3 配置机器人事件与回调¶

    先添加机器人能力。

    然后把“事件配置”和“回调配置”都切到 长连接 并保存。

    4.4 添加事件¶

    原稿对“事件”和“回调”的解释很适合小白：

    1. 事件：飞书里发生的各种事情，例如有人私聊你、有人 @ 你
    2. 回调：飞书把这些事情通知给你机器人的地址
    3. 添加事件：本质上就是订阅你关心的消息类型

    接下来点击“添加事件”，在“消息与群组”里把需要的事件勾上。

    如果一切顺利，这条极简路线到这里就跑通了。

    5. 路线二：本地部署 OpenClaw¶

    如果你想自己掌控模型、工作区和后续扩展能力，那就可以走本地部署路线。

    原稿也很诚实地说了：
    这条路线更难，对代码和命令行的接受度要求更高。

    5.1 Windows 先装 WSL¶

    如果你用的是 Windows，建议先装 WSL，在 Linux 环境里运行 OpenClaw。

    原稿这里的具体命令未展示，所以不在本文补造。
    如果你要完整命令版，可以直接参考本目录里的：

    原稿说明这里可以直接用一条命令完成依赖和 OpenClaw 的安装，但原命令内容在飞书外不可见。

    安装完成后，会出现 OpenClaw 的欢迎界面。

    后面的初始选项，原稿给的是一组经验顺序：

    1. yes
    2. yes
    3. yes
    4. no

    5.3 启动 OpenClaw 配置¶

    原稿这里同样有一条启动命令，但飞书外不可见。
    之后就会进入 openclaw onboard 的配置流程。

    5.4 选择 Kimi 模型¶

    原稿推荐的是 Kimi K2.5，理由是整体性价比较高。

    这里还特别提醒：

    1. .ai 是国际版
    2. .cn 是国内版
    3. subscription 是订阅版

    如果你拿的是 Kimi Code 会员套餐和对应密钥，就要选对订阅版，否则可能会导致没反应。

    5.5 购买 Kimi 会员并获取密钥¶

    Kimi 会员页面：

    https://www.kimi.com/membership/pricing

    Kimi Code 控制台：

    https://www.kimi.com/code/console

    在控制台复制密钥，再填回刚才的命令行里。

    5.6 先跳过飞书渠道¶

    原稿这里建议先不要一上来就接飞书，而是先把模型主线跑通，再回头接渠道。这样更容易排错。

    然后按原稿建议，先安装 skills，再把主流程跑通。

    5.7 验证模型是否跑通¶

    如果出现“网页空白”或“不回复”的情况，原稿给出的经验解法是：
    重新从 openclaw onboard 开始走一遍，必要时重启实例。

    之后可以用 TUI 或 Web UI 测试。

    如果模型已经能正常回复，就说明本地 OpenClaw 主链路跑通了。

    6. 本地 OpenClaw 接入飞书¶

    当本地模型已经跑通后，再开始接入飞书。

    6.1 创建飞书应用¶

    打开：

    https://open.feishu.cn/app

    点击“创建应用”。

    输入应用名称。

    然后记录 App ID 和 Secret。

    接着给应用添加机器人。

    7. 这篇教程适合谁¶

    7.1 适合直接用 Kimi Claw 的人¶

    1. 完全零基础
    2. 先想快速看到结果
    3. 不想自己维护 WSL、依赖和配置文件

    7.2 适合本地 OpenClaw 的人¶

    1. 希望自己掌控模型与工具
    2. 后续要做 Skills、多 Agent、工作区隔离
    3. 接受命令行和配置文件

    8. 结论¶

    如果你只想先把“飞书里的个人 AI 助理”跑起来，优先走：

    1. Kimi Claw

    如果你更在意扩展性、模型控制权和后续玩法，再走：

    1. 本地 OpenClaw + Kimi API + 飞书

    原稿最大的价值其实不是某一条命令，而是这套很实用的节奏：

    1. 先把模型跑通
    2. 再接飞书
    3. 再补权限、事件和回调

    这样排错成本最低，也最适合小白。
